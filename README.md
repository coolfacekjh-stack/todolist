# 📝 Vanilla JS Todo List App

> Vanilla JavaScript + Tailwind CSS로 만든 할 일 관리 앱

---

## 📌 프로젝트 소개

HTML, CSS, JavaScript 파일만을 이용해 만든 간단한 Todo List 앱입니다.  
외부 프레임워크 없이 순수 Vanilla JS로 DOM을 조작하고,  
LocalStorage를 활용하여 브라우저를 새로고침해도 데이터가 유지됩니다.  
AI(Cline)와의 협업을 통해 설계 → 구현 → 리팩토링 순서로 개발하였습니다.

---

## 🎯 학습 목표

- Vanilla JavaScript로 DOM을 조작하는 방법 이해
- 배열(Array) 데이터를 활용한 상태 관리 방법 습득
- localStorage를 이용한 데이터 영속성(Persistence) 구현
- 함수를 최소 단위로 분리하는 코드 설계 방법 학습
- AI와 협업하여 설계 → 코드 생성 → 리팩토링 흐름 경험

## ▶️ 실행 방법

별도의 설치 없이 바로 실행 가능합니다.

```bash

# 2. 폴더로 이동
cd todolist

# 3. index.html 파일을 브라우저에서 열기
# Windows
start index.html

# macOS
open index.html
```

> ⚠️ 인터넷 연결이 필요합니다 (Tailwind CDN 로딩)

## 🔍 AI 생성 결과 검토 기준

AI가 생성한 코드를 받은 후 아래 기준으로 검토하였습니다.

| 검토 항목 | 확인 내용 |
|----------|-----------|
| **동작 확인** | 추가 / 완료 / 삭제 / 새로고침 시 기능이 올바르게 동작하는가 |
| **변수명 명확성** | 변수명만 봐도 역할을 이해할 수 있는가 |

| **주석 가독성** | 초보자가 읽어도 이해할 수 있는 주석인가 |
| **데이터 영속성** | 새로고침 후에도 데이터가 유지되는가 |
| **코드 흐름 일관성** | 데이터 변경 → 저장 → 화면 갱신 순서가 일관되게 지켜지는가 |

## 📚 배운 점

- **설계를 먼저 확실하게 하고 나중에 코드 생성을 요청해야 한다.**  
  UI 구조, 데이터 구조, 함수 목록을 먼저 설계한 덕분에 코드 방향이 명확했고,  
  불필요한 수정 횟수를 줄일 수 있었다.

- **localStorage를 사용해야 데이터가 유지된다.**  
  JavaScript의 변수는 페이지를 새로고침하면 초기화된다.  
  브라우저에 데이터를 영구적으로 저장하려면 localStorage를 활용해야 한다.

- **함수를 잘게 쪼개야 문제 확인과 수정이 쉽다.**  
  하나의 함수가 여러 역할을 하면 버그 위치를 찾기 어렵다.  
  단일 책임 원칙에 따라 작은 함수로 나누면 오류 추적과 유지보수가 훨씬 쉬워진다.

## 📄 3줄 보고서

1. **설계 우선 원칙**: 코드를 먼저 짜는 것보다 화면 구조·데이터 구조·함수 목록을 먼저 설계하면 방향이 명확해지고 수정 비용이 줄어든다.
2. **데이터 영속성**: 브라우저 변수는 새로고침 시 사라지므로, 사용자 데이터를 유지하려면 반드시 localStorage와 같은 저장소를 사용해야 한다.
3. **함수 단일 책임**: 함수를 역할별로 최소 단위로 분리하면 코드 가독성이 높아지고, 버그 발생 시 원인 파악과 수정이 훨씬 쉬워진다.
